# 核心需求 v1

## 智能数据分析与自动化研报生成系统

**英文名称：** Intelligent Data Analysis & Automated Reporting System

---

## 1. 项目概述

本项目旨在构建一个**多智能体协同（Multi-Agent System）**的自动化研究平台。

**核心目标：** 通过输入基础的研究主题（例如：「中国经济发展对中国工业外贸发展的影响研究」），利用 AI 代理接管并自动化执行从**需求定义、深度检索、数据分析框架搭建、代码编写与执行、报告撰写**到**最终工程化排版与交付**的全流程。

**人在环路（Human-in-the-loop）：** 在关键节点引入用户确认，以保证研究方向的准确性与专业度；同时支持通过配置实现**全自动化流转**。

---

## 2. 核心工作流与模块拆解

系统整体架构划分为以下**六个核心模块**，依次串行并支持**状态回溯**。

---

### 模块一：需求捕获与专业定义（Requirements Capture & Definition）

| 维度 | 说明 |
|------|------|
| **输入** | 用户提供的初始研究主题（自然语言文本）。 |
| **处理逻辑** | Agent 接收主题后，启动与用户的多轮对话；通过引导式提问，明确研究的核心变量、时间跨度、目标受众及预期深度。综合对话历史与初始主题，自动生成结构化的《专业需求文档（PRD）》。 |
| **交互机制** | 将生成的 PRD 发送给用户确认；用户可修改、确认，或预先设定为「跳过确认，全自动推进」。 |
| **输出** | 确认版的《研究需求文档》。 |

---

### 模块二：深度检索与研究大纲生成（Deep Research & Outline Generation）

| 维度 | 说明 |
|------|------|
| **输入** | 确认版的《研究需求文档》。 |
| **处理逻辑** | Agent 挂载/调用深度检索（Deep Research）工具或服务；围绕 PRD 与原始主题做多维度信息抓取与文献/资讯检索。基于检索结果自动构建《报告大纲》；将参考资料下载、分类并保存至本地指定文件夹；自动生成 **README.md**，对资料做来源说明与内容概述。 |
| **交互机制** | 与用户多轮对话，对《报告大纲》确认与微调（支持跳过确认）。 |
| **输出** | 确认版的《报告整体大纲》；结构化的参考资料文件夹及其 README。 |

---

### 模块三：数据分析框架拆解与设计（Data Analysis Framework Decomposition）

| 维度 | 说明 |
|------|------|
| **输入** | 确认版的《报告整体大纲》（重点提取实证分析/数据分析部分）。 |
| **处理逻辑** | 针对需数据支撑的章节做专业拆解；**智能规划数据采集路径（核心优化）：**<br>1. **内置 API 推荐（优先）：** 预置多种免费且合规的宏观/金融/产业数据 API（如 AkShare、FRED、世界银行、国家统计局等）；用户不确定来源时，Agent 主动匹配并推荐。<br>2. **用户自主上传（次优先）：** 支持 CSV、Excel 等结构化数据。<br>3. **AI 爬虫（备用）：** 仅在无现成 API 且无用户数据时作为备选；需提示低效与不稳定风险。<br>定义具体分析方法（计量/统计等）、预期产出（表格、图表）及分析逻辑；整理为《数据分析技术大纲》，并形成该阶段参考资料库。 |
| **交互机制** | 多轮对话，用户对《数据分析技术大纲》的数据来源、方法论与逻辑进行确认与调整。 |
| **输出** | 确认版的《数据分析技术大纲》；数据分析阶段专项参考资料文件夹。 |

---

### 模块四：代码自动生成与执行沙盒（Automated Coding & Execution Sandbox）

| 维度 | 说明 |
|------|------|
| **输入** | 确认版的《数据分析技术大纲》。 |
| **处理逻辑** | 将技术大纲转化为编程需求 Prompt；调用 AI 编程模块生成 Python 等脚本；**优先**直接调用预置数据 API（如 `import akshare as ak`），避免冗长爬虫。全自动执行：采集/接口调用、清洗、分析、图表渲染。 |
| **输出** | 可视化图表（PNG、HTML 等）；结构化数据分析结果报告（含核心指标与初步解读）；完整代码（`.py` 或 Jupyter Notebook）；原始数据与清洗后的中间数据。 |

---

### 模块五：深度释义与正文撰写（Deep Interpretation & Content Drafting）

| 维度 | 说明 |
|------|------|
| **输入** | 模块二的《报告整体大纲》及参考资料；模块四的图表与结果报告。 |
| **处理逻辑** | **二次深度检索：** 将数据分析结果（图表趋势、关键数值）回传模型，针对数据结论做二次检索，补充现实支撑、学术解释或政策意义。按大纲分章节生成**撰写提示词（Drafting Prompt）**，再结合资料与数据结果撰写正文（含图表解读、结果文本化及二次检索补充的理论/政策意义）。 |
| **输出** | 各章节完整文本及对应图表占位引用。 |

---

### 模块六：工程化排版与终稿交付（Engineering Formatting & Final Delivery）

| 维度 | 说明 |
|------|------|
| **输入** | 模块五各章节文本及全部图表、附件。 |
| **处理逻辑** | 将文本与图表送入 AI 编程模块（如 `python-docx` 等）自动合并、排版与格式校验；生成图文并茂、逻辑连贯的最终研报（Word 等）。 |
| **输出（最终交付物）** | **最终研报：** Word/PDF；**代码资产、数据资产、资料资产；**过程文档：**《报告大纲》《数据分析技术大纲》；**操作说明：** 总控 README（文件作用、关联与操作提示）。以上内容自动打包为 **ZIP**。 |
